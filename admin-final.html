🔐 Program Upgrade (3-of-5 Gate)

Upgrade Gate PDA: Enter Program ID above

Required. The buffer is a temporary on-chain account that holds the new program .so before upgrade. Create it from your machine (devnet), then paste its pubkey here. Example: solana program write-buffer target/deploy/sanitas_seeker.so --url devnet (note the printed Buffer). Buffer authority must be your gate PDA; if not: solana program set-buffer-authority <BUFFER> --new-buffer-authority <GATE_PDA> --url devnet Note: After a successful upgrade the buffer is emptied (or closed); the loader then returns invalid account data if you reuse it. ⛔ DANGER — Close entire program (recover ProgramData + program account rent)

Do not use this for normal ops. This is not the same as "Recover rent" above (that only closes a buffer).

Closing a program permanently removes the executable at that program id. Clients, PDAs, and users depending on it will break. Use only for an abandoned devnet id or after you have migrated to a new program id.

  • Requires the upgrade authority to sign — usually a keypair in the CLI. If upgrade authority is a PDA (e.g. gate), you cannot run this blindly; you need a governor/CPI path or a temporary authority transfer your governance approves.
  • This page does not send the close tx; it only shows the solana program close command after you pass confirmations.
